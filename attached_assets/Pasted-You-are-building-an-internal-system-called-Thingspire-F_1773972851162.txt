You are building an internal system called "Thingspire Flow".

You MUST strictly follow the Thingspire Design System.

This is NOT optional. All UI must comply with the rules below.

---

# 1. DESIGN PRINCIPLE

- Use token-based design only
- NEVER hardcode colors (no hex values directly in components)
- Use design tokens (primary, secondary, neutral, tertiary, extended)
- Maintain consistency across all screens

---

# 2. COLOR SYSTEM (CRITICAL)

## Primary (Brand Color)
- Used for main actions (buttons, highlights)
- Example usage:
  - primary-400 → main button
  - primary-500 → hover
  - primary-50 → background highlight

## Neutral (MOST IMPORTANT)
- Used for 80%+ of UI
- Includes:
  - backgrounds
  - text
  - borders

## Secondary
- Used for supporting UI elements
- subtle components

## Tertiary (LIMITED USE)
- Used only for emphasis
- MUST NOT exceed 10% of total UI usage

## Extended Colors
- red → error / danger
- yellow → warning
- green → success
- blue → informational

---

# 3. LAYOUT RULES

## MUST use this layout:

[ Sidebar | Main Content ]

---

## Sidebar
- background: dark (neutral-800 ~ neutral-900)
- text: light (neutral-100)
- active menu: primary-400

---

## Main Content
- background: light (neutral-50)
- content area uses cards

---

# 4. CARD DESIGN (DEFAULT UI UNIT)

All content must be inside cards.

Card style:
- background: white
- border: neutral-200
- border-radius: 8px ~ 12px
- subtle shadow using neutral opacity

---

# 5. BUTTON RULES

## Primary Button
- background: primary-400
- text: white
- hover: primary-500

## Secondary Button
- background: neutral-100
- text: neutral-800

## Danger Button
- background: red-500

---

# 6. SURVEY UI RULES

## Survey Intro Page
- clean layout
- background: neutral-50
- highlight box: primary-50

## Anonymity Notice Box (IMPORTANT)
- background: primary-50
- left border: primary-400
- text: neutral-900

## Question Card
- use card layout
- question text: neutral-900
- description text: neutral-600

## Likert Scale Buttons
- default: neutral-200
- hover: primary-200
- selected: primary-400
- selected text: white

## Text Input (Textarea)
- border: neutral-300
- focus: primary-400
- background: white

---

# 7. DASHBOARD RULES

## Score Visualization
- high → green
- medium → yellow
- low → red

## Charts
- only use:
  - primary
  - secondary
  - neutral
- DO NOT overuse tertiary colors

---

# 8. STRICT PROHIBITIONS

- NO random colors
- NO inconsistent spacing
- NO different styles for same components
- NO direct hex values in UI components

---

# 9. UX TONE

- clean
- minimal
- professional
- calm (NOT flashy)

---

# 10. FINAL RULE

If unsure:
- default to neutral colors
- use primary for emphasis

---

Follow this design system strictly when generating UI components.
